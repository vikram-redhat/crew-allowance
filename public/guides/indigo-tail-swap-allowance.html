Tail Swap Allowance for IndiGo Pilots — When You Get Paid ₹1,500

PAH FLT Issue 01 Rev 46 §6.0 · Last verified 2 May 2026 · Reading time ~5 min

IndiGo's tail swap allowance pays ₹1,500 per qualifying swap to Captains and ₹750 per qualifying swap to First Officers. A "tail swap" is a change of aircraft (different registration) between two consecutive sectors within the same duty period. The allowance recognises the additional pre-flight setup work needed when the aircraft changes mid-duty.

The rule in one sentence

For every consecutive sector pair within the same duty, if the aircraft registration differs and the pair isn't excluded by the DHT/DHF→DHF rules, pay ₹1,500 (Captain) or ₹750 (First Officer).

The rate (effective 1 January 2026)

RankPer qualifying swap
Captain₹1,500
First Officer₹750

Senior Captain, LTC, TRI and DE roles use the Captain rate. Senior First Officers use the First Officer rate. The allowance is a flat per-swap amount, not prorated to time.

Eligibility — which combinations pay

Within a single duty period, three sector-pair combinations qualify:

  • Operating → Operating. Both sectors are normally flown by the pilot, on different aircraft.
  • Operating → DHF. The pilot operates the inbound, then deadheads forward on a different aircraft.
  • DHF → Operating. The pilot deadheads in on one aircraft, then operates a different one.

Two combinations do not qualify:

  • Any pair involving a DHT sector on either the inbound or outbound side. DHT (deadhead-terminating) is excluded under PAH §6.0.
  • DHF → DHF. A pure out-and-back positioning round-trip — even if the two aircraft are different — does not pay.

How a swap is detected

The trigger is the aircraft registration (e.g. VT-IXX vs VT-IYY), not the aircraft type. CrewAllowance.com retrieves the registration for each sector from FlightAware AeroAPI for the operation date. If the registrations of two consecutive same-duty sectors differ — and the pair isn't excluded — that's one swap.

Same type can still swap. A change from one VT-Ixx A320 to another VT-Iyy A320 is a tail swap — same aircraft type, different registration. Type doesn't matter; registration does.

Worked example

Captain E. — March, BLR base — three duties, two qualifying swaps

Captain E. has three multi-sector duties in March. Each duty has at least one inter-sector pair worth checking:

  1. Duty 1 (07 Mar): BLR → DEL (VT-IXX, op) · DEL → COK (VT-IYY, op).
    Different registrations, both operating, same duty. Qualifying swap — ₹1,500.
  2. Duty 2 (14 Mar): BLR → BOM (VT-IZZ, op) · BOM → DEL (VT-IZZ, op).
    Same registration on both legs — no swap. ₹0.
  3. Duty 3 (22 Mar): BLR → DEL (VT-IAA, DHF) · DEL → MAA (VT-IBB, op) · MAA → BLR (VT-IBB, DHT).
    Pair 1: DHF → Op, regs differ. Qualifying — ₹1,500.
    Pair 2: Op → DHT, excluded by DHT rule. ₹0.
March tail swap total = ₹1,500 + ₹0 + ₹1,500 + ₹0 = ₹3,000

Captain E. should see ₹3,000 on the "Tail Swap Allowance" line of their March payslip. A First Officer flying the same three duties would see ₹1,500.

The midnight-delay edge case (a real source of phantom swaps)

Why a delayed flight can look like a tail swap when it isn't

When a sector scheduled for the late evening is delayed and takes off after midnight, the airline's flight-status API may return the next day's rotation when queried with the actual takeoff date — a different aircraft, doing the same flight number. That can produce a phantom tail swap that wasn't really there: the aircraft you flew was actually the previous day's tail, the API just looked up the wrong day.

Example: flight 6E6846 STV → DEL is scheduled to depart at 21:30 on 06 March. It runs late and actually pushes back at 00:11 on 07 March. If the calculator queries the API for "6E6846 on 07 March", it gets the 07 March rotation — a different tail, VT-IZZ. The flight you actually flew was the 06 March rotation on VT-IXX, the same aircraft as your inbound 6E6845 STV-arrival earlier that evening. There was no swap.

CrewAllowance.com detects this pattern: PCSR ATD in the early-morning window (00:00–06:00) but the schedule API's ATD elsewhere. When detected, it re-queries the previous day's schedule and uses that registration instead. Affected rows in your breakdown carry a small "↺ shifted" badge so you know the date-shift was applied.

Common reasons your tail swap allowance looks wrong

1. A phantom midnight-delay swap was counted

Described above. If your tail swap total has one extra ₹1,500 (Captain) and a duty that ran past midnight is suspect, this is the most likely cause. The calculator's date-shift detection should catch it; if it doesn't, surface the affected duty for manual review.

2. A DHT sector wasn't flagged in the PCSR parser

DHT sectors are excluded from tail swap pairing. If a DHT flag in the Other Crew section was missed, an excluded pair may have been counted as a swap.

3. Same-registration pairs were counted as swaps

If the schedule API returned slightly different registration formats (e.g. "VT-IXX" vs "VTIXX") for the same aircraft, a non-swap can look like a swap. CrewAllowance.com normalises registration strings before comparing.

4. Pairs across different duties were counted

Tail swap allowance applies only within a single duty. A change of aircraft between two separate duties — typically separated by a layover or end-of-day — does not pay. The duty boundary is what makes the difference.

5. An "unverified" pair was treated as a swap (or as a non-swap)

If the schedule data provider returns a null aircraft registration for either side of a pair, the calculator surfaces the pair as unverified rather than guessing. Both miscounting directions are possible — surfacing keeps it transparent.

Frequently asked questions

What is the tail swap allowance for IndiGo pilots?

IndiGo's tail swap allowance under PAH §6.0 pays ₹1,500 per qualifying swap to Captains and ₹750 per qualifying swap to First Officers. A "tail swap" is a change of aircraft (different registration) between two consecutive sectors within the same duty period.

What counts as a tail swap?

Two consecutive sectors within the same duty flown on aircraft with different registrations. If you operate flight A on VT-IXX, then operate flight B on VT-IYY without a layover or end-of-duty between them, that is one tail swap — ₹1,500 (Captain) or ₹750 (First Officer).

Which sector combinations qualify for tail swap allowance?

Three combinations within the same duty qualify: operating to operating, operating to DHF, and DHF to operating. Two combinations are excluded: any pair involving a DHT sector on either side, and DHF to DHF (a pure positioning out-and-back).

How is a tail swap detected?

By comparing the aircraft registration of two consecutive sectors within the same duty. If the registrations differ and the pair isn't excluded by the DHT/DHF→DHF rules, it counts as a swap. CrewAllowance.com retrieves registrations from FlightAware AeroAPI for the operation date.

What is the midnight-delay edge case for tail swaps?

When a flight scheduled for the late evening is delayed past midnight, the airline's flight-status API may return the next day's rotation — a different aircraft — when queried with the actual takeoff date. This can produce a phantom tail swap that wasn't really there. CrewAllowance.com detects the pattern (PCSR ATD in 00:00–06:00 plus an asymmetric API result) and re-queries the previous day's schedule.

Does the tail swap allowance pay if the aircraft type changes?

The PAH §6.0 trigger is the registration difference, not the aircraft type. A swap from one VT-Ixx A320 to another VT-Iyy A320 still pays — same type, different registration. A swap to a different type (A320 to A321) also pays — type doesn't matter, only registration.

Why does CrewAllowance.com show some pairs as 'unverified'?

If the schedule data provider returns a null aircraft registration for either sector of a pair, the calculator can't determine whether registrations matched. Rather than guess, it surfaces the pair as unverified — neither counted as a swap nor counted as a non-swap. This is conservative and lets you cross-check those specific pairs against your own records.

Do tail swaps stack — multiple swaps in one duty?

Yes. Each qualifying inter-sector pair within a duty pays separately. A duty with three sectors flown on three different aircraft (A → B → C, all different registrations) earns two swaps — one for the A-to-B pair, one for the B-to-C pair — totalling ₹3,000 for a Captain.
